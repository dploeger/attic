Fiddling with Funambol

I'm currently fiddling with Funambol, which should be known as “Sync4j” to some of you. Funambol's a plattform-, device- and backend-independent synchronisation plattform. As these three features are certainly a good thing, the bad side of it is: it's done in Java.

Bah. I don't like Java very much. Especially for web services (which Funambol is after all).

Another bad side is the bad documentation for its features and the configuration. Well, maybe it is bad, or it's documentation for Java programmers and I just don't get it. Could be aswell.

Well, now I'm pumping my brain with long lost Java programming experience to write a SLOX (SuSE Linux Open Exchange Server, better grandfather of OpenXchange)-Connector. Basically SLOX's just a bunch of Tomcat-Controlled (Bah! Java again) data in a PostgreSQL-database, an OpenLDAP-Server and a Cyrus IMAP-Backend. So the data shouldn't be that hard to sync. I'm gonna try a one-way sync first though, because I think the days of SLOX at work are numbered.

So I'm using the spare documentation around Funambol to create myself a JDBC-based connector to the PostgreSQL. Though it's written in Java, the basic blocks work quite good right now. I'll post the connector under the GPL here when I'm ready.
