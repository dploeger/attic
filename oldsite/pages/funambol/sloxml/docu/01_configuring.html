2. Configuring

Introduction

To configure the real synchronization data source with funambol, start the funambol administration tool and double click on “Funambol Administration tool” to connect to your funambol server. Accept the defaults (or enter the real username/password if you've already configured your funambol-server) and login.

Creating a new data source

Complete these steps to create your sloxml data source:

  • Go to Modules/sloxml/FunambolSloxMLConnector/SloxML SyncSource
  • Right click and select “Add SyncSource”
  • In the right pane enter the following values:
    • SourceURI: That's the name of the data source, that you'll enter in your mobile device later on. (i.e: ./sloxmlc)
    • Name: The real name of the data source. I haven't figured out yet, what it means. (i.e.: sloxmlc)
    • Type: Enter “SIF-C” for contact synchronisation, “SIF-E” for calendar synchronisation. (i.e.: SIF-C)
    • JNDI Name of SLOX-Database: Enter the JNDI-name for the slox-database, that you've created using installation. (i.e.: jdbc/sloxmlds)
    • Hostname/IP of the LDAP host: Enter the hostname (or IP) of the LDAP host of your SLOX-installation. This is mostly the server, the rest of SLOX runs on.
    • Port of the LDAP Database: The port of your LDAP-Server (normally 389)
    • BaseDN of the LDAP Database: The base-DN of your LDAP-database. (That's per SLOX-default your domain in the form: dc=company,dc=com)
    • DN of the user to bind with: Specify a user, that can read your LDAP-Database 1)
    • Password of the user to bind with: Specify the password of the user given above.
  • Click “Add”

Using the examples provided here, you've created a sloxml sync source for synchronising your contacts. You can now create another sync source (i.e. ./sloxmle) for calendar synchronisation using the type SIF-E.

Configuring text encoding

For good transportation over the HTTP-layer, which funambol uses as the main transport-layer for the synchronisation, you'll have to base64-encode all your data, that is transported.

  • Doubleclick on “Server settings” and then click on “Configure” beside the value for “Data transformer manager”.
  • In the frame “Data transformation”, click on the ”+”-button
  • In the left column of the new row enter your sloxml-URI (i.e.: ./sloxmlc). In the right column enter “b64”. (You have to doubleclick into the cell to activate editing)
  • Click “Save”

Note: You'll have to replay these steps for each SloxML-based sync source you create!

Configuring synchronisation strategy

Currently sloxML only supports a one-way sync from the server to the client, meaning you cannot add new items on your mobile device and sync them back to the server. Perhaps someone embeds this in the source and sends it back to me, so I can embed it here.

To set the right strategy for this restriction complete these steps:

  • Doubleclick on “Server settings” and then click on “Configure” beside the value for “Strategy”.
  • Select “Server wins” in the row described as “sloxml (./sloxml)”
  • Click “Save”

Note: You'll have to replay these steps for each SloxML-based sync source you create!
