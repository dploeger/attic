Description

Many companies use LDAP-Directories for several reasons (i.e. online addressbooks in company telephones). Zimbra currently offers no possibilty to export contacts into a LDAP-Directory.

This script fetches addressbooks in VCF-format and generates a LDAP-tree out of it. It should be used on a separate LDAP-Server. It could work on the Zimbra-Server itself, though this is heavily NOT recommended.

@Zimbra: If you'd like to integrate it in main Zimbra, go ahead! It would all make us happy :)

How it works

zmPublish is a small Perl-Script, that should be called with a configuration file. In this configuration file, the admin can setup so-called “publishers”, that fetch a Zimbra addressbook in VCF-format, maps the VCF-fields to LDAP-attributes and then builds up a LDAP-tree in the configured LDAP-server.

Installation

Just unzip the file and configure the zmpublish.cfg to address you needs. The configuration file is very self-explanatory.

FAQ

  • Why does my LDAP tree has elements like uid=1, uid=2, uid=3 and so on?
    • I didn't put the cn-attribute into the dn, because then we would have problems with people like john miller or something
  • The script says, that inetorgperson needs an attribute called sn. What about it?
    • Please check the entry in your Zimbra addressbook to have a surname. The inetorperson-schema, that the script uses in the LDAP-server needs this data.
